API Documentation

This API provides access to user, expense, and income data. Below are the available endpoints.

General Endpoint

GET / - Returns a brief description of the API and available endpoints.

Users Endpoints

  • GET /users - Retrieve all users.
  • POST /users - Add a new user.
  • PUT /users/:id - Update an existing user by ID.
  • DELETE /users/:id - Delete a user by ID.

Expense Endpoints

  • GET /expenses - Retrieve all expenses.
  • POST /expenses - Add a new expense.
  • PUT /expenses/:id - Update an existing expense by ID.
  • DELETE /expenses/:id - Delete an expense by ID.

Income Endpoints

  • GET /income - Retrieve all income.
  • POST /income - Add a new income.
  • PUT /income/:id - Update an existing income by ID.
  • DELETE /income/:id - Delete an income by ID.

PUT requests

For PUT requests, the request body should contain only the fields that need to be updated. For example, to update the name of a user, the request body should contain the name field only.
